Project Proposal


Written Description


This project will provide a database of resources to aid players through the new Nintendo title “Pokémon Legends: Arceus.” In a world where the popularity of video games is ever-growing, so too is the demand for new databases where players can do their own research to help them better understand games or discover new things about them. The goal of this project is to provide a simple, straight-forward, accessible database to give players new and old a chance to learn all about the newest title (released January 28, just two weeks prior to today’s date). Since the core mechanics of this game are so vastly different than its predecessors, existing database models have become archaic. This could be a great opportunity to get ahead of the market and provide an innovative way to access all the Pokémon Legends: Arceus data one could ever need.


Data Requirements


For each Pokémon, it is necessary to have:

  • Pokédex number (primary key)

  • Biological sex

  • Combat type (at least one, at most two)

  • Preferred types of food

  • Average weight

  • Different forms (shiny, male, female)

  • Behavior (aggressive vs. passive)

  • Items dropped

  • Current level (or average spawning level)

  • “Nature” (innate trait that prioritizes certain stats)

  • Combat stats (health, attack, special attack, defense, special defense, and speed [varying with different levels/natures])

  • Locations found

  • Evolutions available

  • Innate Learnset (attacks or abilities each Pokémon gains while leveling up)

  • Taught Learnset (attacks or abilities each Pokémon is capable of learning from a trainer)

  • Effort values (base stat upgrade levels)


For each attack or ability, it is necessary to have:

  • Move ID (primary key)

  • Combat type (one only)

  • Damage dealt (could be zero)

  • Stat-altering effects (could be zero)

  • Status effects (could be none)

  • Accuracy rating

  • Battle style (strong vs. agile [this affects the overall effect of every move])

  • PP (Poké Power [the number of times a move can be used])


For each combat type, it is necessary to have:

  • Type ID (primary key)

  • Moves of said combat type

  • Pokémon of said combat type

  • Type matchups, which can affect the effects and/or damage of moves (example: Fire)

    • Offense – Weak: Fire, Water, Rock, Dragon

    • Offense – Neutral: Normal, Ground, Fairy, Electric, Fighting, Psychic, Dark, Ghost, Flying, Poison

    • Offense – Strong: Grass, Bug, Ice, Steel

    • Defense – Weak: Water, Rock, Ground

    • Defense – Neutral: Normal, Electric, Fighting, Psychic, Dark, Ghost, Flying, Poison, Dragon

    • Defense – Strong – Grass, Bug, Ice, Steel, Fairy

    • No Effect: None


For each item in the game, it is necessary to have:

  • Item ID (primary key)

  • Item type (healing, thrown, crafting)

  • Item effects

  • Item capacity (how many can be held at one time)

  • Item locations (where to obtain items)

  • Item cost (if applicable)

  • Item potential crafting recipes (if applicable)


For each location in the game, it is necessary to have:

  • Location ID (primary key)

  • Biome (type of climate)

  • Naturally occurring Pokémon

  • Naturally occurring items (minerals, herbs for crafting)

  • Rest camp locations (places to shop, rest, and change Pokémon)







Operations


Our website’s operations will allow the user to search for Pokémon, combat moves, items, and in-game locations within our database using keywords and tagged phrases. If the user searches for a particular Pokémon, the user is shown various pieces of information regarding that Pokémon including base stats derived from various factors that influence them from the user’s chosen level, combat nature, and effort values. They can also access images of different forms for each Pokémon, their combat types, tables housing innate and taught learnsets (according to the level at which moves become available), interactable items, evolutionary lineage, locations found, and so much more. Discover all there is to know about the Hisui region’s Pokémon here!




Business Rules


Some essential rules of this iteration of Pokémon include:


  • Every Pokémon must have at least one combat type but can have at most two.

  • Every Pokémon is capable of learning new “moves” (attacks or abilities).

  • Every Pokémon innately has at least one move.

  • Each Pokémon can have at most four active moves in their skillset. However, they may reserve several inactive moves to swap at the user’s discretion.

  • Each move must have and may only have one combat type.

  • Each move may have at least one effect (damage dealt, stat alterations) but can have many. However, they are also capable of having no effect whatsoever or different effects depending on the Pokémon’s combat type.

  • Each Pokémon can interact with several different items.

  • Each item may have a particular location in which it can be found but can have none.

  • Each item may have a particular sell price but can have none.

  • Every location on the map can have several unique Pokémon that spawn there.

  • Every location on the map can have several unique items that spawn there.

  • Every location has at least one biome but can have several, which can have an effect on Pokémon’s moves depending on the combat type (rain enhances water-type moves, sunlight enhances grass-type and fire-type moves, etc).
